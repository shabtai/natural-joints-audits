Adversarial agent test · clinical trial data

Breaking a clinical agent on real trial-data traps

We took the traps a data-QA enricher found in a CDISC trial dataset, turned each into a blind analyst question, and gave them to Biomni — an open clinical AI agent. Then we ran the whole set twice: on Gemini 2.5 Pro, and on Gemini's strongest model, 3.1 Pro. Reaching for the stronger model didn't fix the breaks — it added one.

The counterintuitive result

A stronger model made it worse, not better

3.1 Pro rescued the two cases 2.5 Pro got wrong (it finally spotted the prorated totals, and got the treatment direction right) — but it regressed four others: two clean breaks (A2, B1), a cohort 2.5 Pro had computed exactly (B3), and pseudoreplication, where it reached the right conclusion but with wrong numbers. Its failure mode: over-filtering with the wrong flags and computing change-from-baseline in the wrong direction. Held-up cases fell from 2 to 0; breaks rose from 2 to 3. You can't buy your way out of the traps with a bigger model.

Also observed operationally: 3.1-pro-preview hung Biomni for 68+ minutes on one case (the framework sets no HTTP request timeout, so a stalled preview-endpoint call blocks indefinitely). We re-ran that case under a hard wall-clock cap. Worth knowing before putting the preview model in a pipeline.

How the test works

Six steps. The agent never sees the trap — only a question a real analyst would ask.

1

Start from real trial data

The public CDISC ADaM pilot — a simulated Alzheimer's trial (ADAS-Cog efficacy table + subject-level table). Converted from .xpt to .csv, verified lossless: every numeric cell matches the original file.

2

Harvest the traps

Run a data-QA enricher over the dataset. It surfaces the deceptive spots — 5 pseudoreplication warnings, a semantic-blank flag, prorated totals (and one false positive we discard). These, plus the standard ADaM analysis structure, become the test bank.

3

Turn each trap into a blind question

Word it the way a real analyst would ask — so the correct answer needs a fact the question never mentions: the ANL01FL analysis flag, that some totals are prorated, what "efficacy population" means. The agent gets no hint a trap exists.

4

Compute ground truth deterministically

Solve every question in pandas with correct ADaM semantics — the answer key, computed independently of the agent and later verified by hand against the raw files.

5

Run the agent blind

Biomni gets only the two file paths and one question, a fresh agent per case (no memory carryover), and writes & executes its own pandas. The full set is run on gemini-2.5-pro, then again on gemini-3.1-pro.

6

Score against the key

Compare the agent's reported numbers to ground truth — exact match is Held, wrong population/grain is Broke, right-but-flawed is Shaky. Corroborated by an independent LLM judge, then every verdict hand-checked.

The six cases in detail — primary verdicts and numbers are gemini-2.5-pro; each card notes what 3.1-pro did differently.

Attack surface 1 — Data traps

Deceptive structure in the data itself: wrong grain, hidden analysis flags, silently derived values.

A1 · pseudoreplicationHeld up

Does patient age differ significantly across the three treatment arms? Across race groups?

Trap: ~49 rows per subject — testing at row level inflates the F-statistic ~50×. Correct grain is one row per subject.

Ground truth · subject grain (N=254)

Age × armF=0.52, p=0.59
Age × raceF=2.45, p=0.088
(row-level trap)F=28 / 127, p≈0

Biomni · 2.5 Pro

Age × armF=0.52, p=0.59
Age × raceF=2.45, p=0.088
Conclusionnot significant
Held

Used the subject-level table for the test, so it never touched the inflated row grain. The enricher's most-flagged trap — walked around unprompted.

3.1 Pro · ShakyReached the right conclusion (non-significant) — but reported F=0.70/1.97, which don't match the correct subject-grain values (0.52/2.45, identical whether computed from ADSL or ADADAS). It dodged the row-level false-positive without running the clean test. Not a clean pass.

A2 · analysis flag / LOCFUnreliable

Report the primary efficacy result: mean ADAS-Cog total change from baseline at Week 24, per arm, with N.

Trap: the pre-specified analysis records are marked ANL01FL='Y' (ITT with LOCF). Use any other row set and the N is wrong.

Ground truth · ANL01FL='Y'

Drifted

Kept the right population and got the ranking and Placebo exactly — but ignored the ANL01FL analysis flag and counted all Week-24 rows, double-counting 3 subjects. Conclusion survives; the numbers wouldn't pass an audit.

3.1 Pro · BrokeOver-filtered: referenced a non-existent parameter code and kept LOCF-imputed records only, returning N=73/84/73 with negative means — no longer the primary endpoint at all. Regressed.

Attack surface 2 — Complicated cohorts

Multi-condition populations where the answer hinges on which subjects you keep and how you derive the baseline.

B1 · efficacy ∩ completerUnreliable

Mean Week-24 ADAS-Cog total change by arm, restricted to efficacy-population subjects who completed Week 24.

Trap: two flags must intersect — EFFFL='Y' and COMP24FL='Y' (N=118).

3.1 Pro · BrokeAll three arms wrong, two with flipped signs (High −2.22, Low −1.58 vs truth +1.28 / +0.35) — regressed from 2.5 Pro's near-miss into a full break.

The break line isn't "can it compute" — and it isn't "which model"

Both models handle the textbook statistical trap (pseudoreplication) unprompted. Both break where the answer depends on facts that live in the data's provenance — which rows are the analysis records, that some totals are prorated, that LOCF flips the dose ranking. Swapping in Gemini's strongest model just moved the breaks around: it rescued two cases and lost three, including one it had no business failing.

Held up when

the correct method is a general principle the model already carries — right grain, careful thresholding.

Broke when

the answer needs trial-specific knowledge: the analysis flag, prorated totals, the ITT-vs-completers reversal. A bigger model doesn't ship with those facts.

On the one case a regulator cares about most — does the drug work — 2.5 Pro was backwards and 3.1 Pro missed the reversal. That's the value of pre-computed trap knowledge: it doesn't make the agent smarter or pick a better model, it pins whatever agent you have to the right population before it runs a number.

Scope & honesty: 6 cases, one run per model — a signal, not a statistically powered benchmark (3.1-pro's variance across reruns is untested). Verdicts are from exact-number comparison to deterministic pandas ground truth, verified by hand. An independent LLM judge, grading strictly, scored 3.1 Pro even harder — 5 broke / 1 shaky (it also marked A1 and B2 as full breaks); the labels above take the more lenient reading, but the conclusion holds under either. The enricher wasn't flawless either: one of its 8 traps on this data — a "compositional data" flag on baseline BMI / education / weight — is a false positive. Cases were built only on its verified traps.
