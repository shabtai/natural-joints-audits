Agent benchmark · clinical & biomedical data

We tried to break a biomedical AI agent. Here is exactly where it broke — and where it didn't.

We ran Biomni — an open-source biomedical AI agent from Stanford (3.3k★) — on 21 blind analysis questions across three data domains. Each question was built from a real data trap and scored against deterministic ground truth we computed by hand. The agent is a strong data scientist. It fails in one specific, predictable place: the opaque conventions of standardized clinical-trial data.

The finding

What actually predicts failure

Not "can it compute" — and not how obscure the data is. It's whether the right answer is an anomaly to spot or an opaque convention to follow.

Biomni handles general data-integrity work well, even on datasets it has never seen: it caught an Excel gene→date corruption in an obscure neuroblastoma table, flagged two byte-identical duplicate tables in a Rome microbiome study, and knew that only 978 of 12,328 LINCS genes are directly measured. It aggregated 584,944 single cells to the donor level unprompted to avoid pseudoreplication. These are anomalies and well-documented conventions — things a good analyst, or a well-trained model, recognizes.

It breaks where the correct answer depends on an opaque, dataset-specific convention that looks like ordinary data: that ANL01FL='Y' marks the one analysis record per subject (with LOCF imputation already applied), that ACTOT is an 11-item subscore and not the sum of all 14 items, that a time-to-event median must be censoring-aware. Nothing looks wrong — you simply have to know the rule. That knowledge is niche and regulatory, and the model doesn't carry it.

How the test works

Six steps. The agent never sees the trap — only a question a real analyst would ask.

1

Start from real, public data

The public CDISC ADaM pilot (a simulated Alzheimer's trial), the Human Lung Cell Atlas, and three niche omics datasets. All conversions were verified lossless: every numeric cell matches the source file.

2

Harvest the traps automatically

A semantic data-QA enricher profiles each dataset and surfaces its deceptive spots — hidden analysis flags, prorated totals, pseudoreplication, duplicate tables. The traps come from the tool, not from cherry-picking.

3

Turn each trap into a blind question

Phrased the way a real analyst asks — so the correct answer needs a fact the question never states. The agent gets no hint a trap exists.

4

Compute ground truth deterministically

Every question is solved independently in pandas with correct domain semantics, then re-verified by hand against the raw files before publication.

5

Run the agent blind

Biomni gets only the file path and one question, a fresh agent per case (no memory carryover), and writes & executes its own pandas.

6

Score against the key

The agent's reported numbers are compared to ground truth: exact match = Pass, wrong population/grain/derivation = Break, right-but-flawed = Partial. An independent LLM judge corroborates; every verdict is hand-checked.

Why it fails — from the generated code

Not arithmetic. Every wrong answer traces to one thing: the agent re-derives what the data already provides.

CDISC ADaM is analysis-ready by design: ANL01FL='Y' marks the exact record to use (LOCF already applied), CHG and BASE are the pre-computed change and baseline, ACTOT is a specific named parameter. You are meant to use these columns. The agent instead rebuilds the statistics by hand — and the re-derivations are where it breaks. The code is its own.

B2 · 2.5-proHand-rolled the change & LOCF, skipped the analysis flag
# ignores the ready-made CHG column and the ANL01FL flag
itt['CHG'] = itt['AVAL'] - itt['BASELINE']            # recompute change
last = itt.loc[itt.groupby('USUBJID')['AVISITN'].idxmax()]  # DIY LOCF
# never filters ANL01FL == 'Y'  →  effect = +3.62  (truth: −1.04)
Consequence: its self-built baseline and LOCF ran over the wrong record set and inverted the treatment effect.
A3 · 2.5-proMisread what the total score is
items = adadas[adadas['PARAMCD'] != 'ACTOT']          # all 14 item scores
calc  = items.groupby(['USUBJID','AVISIT'])['AVAL'].sum()
# but ACTOT is the "Adas-Cog(11) Subscore" — 11 items, not 14
# verified: 0% of complete visits match (sum of 14 = 42, ACTOT = 13)
Consequence: the reconstruction disagreed with every record, so it flagged all 818 totals instead of the ~33 genuinely prorated ones.
A2 · 3.1-proHallucinated a parameter code
filtered = adadas[adadas['PARAMCD'] == 'ACT0111']   # invented code
# mangled from the label "Adas-Cog(11)"; the real code is 'ACTOT'
# then thrashed across DTYPE / ITTFL / EFFFL / LOCF combinations
Consequence: final numbers match no valid analysis population.
The stronger model was not reliably better. Across the clinical set, gemini-3.1-pro rescued two cases but regressed three — and it thrashed: it tried more flag combinations without knowing which is authoritative. Reasoning harder does not substitute for knowing the standard.

A separate red flag — answers that contradict the code

While hunting for failures on the preview model, we found a reliability issue distinct from domain knowledge.

On gemini-3.1-pro (preview), Biomni repeatedly wrote correct code and then reported a final number untethered to it. This is not a convention error — the agent knew the convention — it is an orchestration/output-reliability failure of the agent×preview-model integration (the same pairing that also hung for 68 minutes on one run).

H1 · 3.1-proCorrect code, disconnected answer
creat[creat['AVISIT']=='Week 24'].groupby('TRTP')['CHG'].mean()
# this code output: Placebo −17.68, High −4.42
# but the agent REPORTED: Placebo −0.55, Low +1.17, High +1.05
Also seen on C3 (code computed 92, answer said 21; correct on re-run) and C5 (a correct value_counts(), wrong reported number). For a scientific agent, a final answer that silently disagrees with its own computation is arguably worse than a domain error — and it means some "breaks" on the preview model are reliability, not knowledge. The clean convention breaks (B2, A3, C2, H2) do not depend on this.

The competence boundary is domain provenance, not data science

Biomni is a capable analyst. It sidesteps textbook statistical traps unprompted, catches corrupted and duplicated data on studies it has never seen, and reproduces exact cohorts. The wall it hits is specific and repeatable: the opaque, documented conventions of standardized regulated data — which record is the analysis record, how a derived value was built, what an assay actually measures.

Held up when

the task is an anomaly to spot or a well-documented method to apply — duplicates, corrupted genes, pseudoreplication, landmark genes, censoring direction. General data-science competence, plus conventions the internet has written about.

Broke when

the answer requires an opaque provenance rule the model can't infer from the data: the analysis flag, the LOCF-imputed record, the subscore definition, the censoring-aware estimator. Niche, regulatory, under-documented — and load-bearing.

This is a narrow but sharp conclusion. A bigger model does not fix it (the strongest one broke more). Obscurity of the dataset does not cause it (obscure omics passed). What fixes it is supplying the convention — pinning the agent to the right record, the right derivation, the right estimator before it computes. That is exactly the knowledge a domain-aware data layer can carry, and exactly where an unaided agent is most confidently wrong.

Scope & honesty. 21 cases, one run per model — a signal, not a statistically powered benchmark; run-to-run variance is untested. Verdicts are from exact-number comparison to deterministic pandas ground truth, hand-verified against the raw files, and corroborated by an independent LLM judge (which graded the strong model even harder than shown here). All data is public. C4's "correct" answer is itself convention-ambiguous, and two of the strong-model breaks are entangled with the code-vs-answer reliability bug — both are flagged in place rather than folded into the headline. The enricher that sourced the traps was not flawless either: one of its flags on the single-cell data (a "compositional" reading of BMI / education / weight) is a false positive, and was discarded. Cases were built only on verified traps.
