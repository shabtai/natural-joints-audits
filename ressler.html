NeoBCC — Audit Report

Generated: 2026-04-30  |  Dataset: 12 post-treatment basal-cell-carcinoma scRNA-seq samples (GSE268728)  |  Domain: Neoadjuvant T-VEC (oncolytic HSV-1) in basal cell carcinoma  |  Code: cancerbits/ressler2024_neobcc  |  Data: GEO GSE268728  |  Paper: Ressler et al., Nature Cancer (2025)
Audit by Natural Joints
Audit design: Multi-agent audit covering code, data honesty, and data facts, followed by independent verification, public-record novelty checking, and impact quantification. The full audit produced a larger set of findings; this report covers only those whose impact on the paper’s reported results is rated MAJOR or MODERATE.

Headline: The headline n=2 herpes-reactive T-cell claim and the n=4-vs-4 Wilcoxon comparisons in Figure 4 and Extended Data Figures 9–10 sit on top of (a) a hardcoded clinical-response group assignment that disagrees with the metadata file shipped in the same repository for 4 of 12 patients, (b) a cell-barcode renaming regex that silently drops 9.6% of cells from the clonotype joins, (c) Wilcoxon tests run at the mathematical floor of the test, and (d) silent NA propagation that drops a patient from a critical n=4 comparison. Several of these compound on the same panels.
Reproducibility caveat. The R pipeline cannot be rerun end-to-end from public materials — the manual cluster-annotation file (Ressler2024_metadata.csv) referenced by INIT_Ressler2024.Rmd:492 is not in the GitHub repository or on GEO, and the recommended preprocessed Seurat object (20241005_Ressler2024_NeoBCC.Rds) is not on GEO either. Impact ratings below are therefore established by structural proof — citing specific code lines, the deposited data values they run against, and the deterministic consequence — rather than by re-execution. Where the paper’s rendered notebooks (notebook/*.md) preserve intermediate output, that output is used as ground truth.

Major findings

Finding 1 — Hardcoded clinical-response groups disagree with the repo’s own metadata file for 4 of 12 patients
VERIFIED NOVEL* MAJOR
Rule typeHardcoded label vector contradicts authoritative metadata Locationnotebook_template/ExtendedData_Figure9.Rmd:68–72; identical block in ExtendedData_Figure10.Rmd:71–75; truth file metadata/ADDITIONAL_METADATA.txt

Both notebooks construct the WHO clinical-response label by hardcoded patient list:

meta$WHO <- "SD"                                                                   # default
meta$WHO[meta$patient %in% c("NeoBCC007_post","NeoBCC008_post","NeoBCC012_post","NeoBCC017_post")] <- "CR"
meta$WHO[meta$patient %in% c("NeoBCC004_post","NeoBCC006_post","NeoBCC010_post","NeoBCC011_post")] <- "PR"

The truth file metadata/ADDITIONAL_METADATA.txt, two directories away in the same repository, disagrees on the PR vs SD assignment for four patients:

PatientMetadata (truth)Hardcode (used)Match
NeoBCC005PRSD✗
NeoBCC010SDPR✗
NeoBCC011SDPR✗
NeoBCC015PRSD✗

The structural consequence is severe. Counting only patients used in the WHO-stratified analyses:

  • The “PR” group as analyzed = {004, 006, 010, 011} — 50% true PR + 50% true SD
  • The “SD” group as analyzed = {005, 014, 015, 018} — 50% true PR + 50% true SD
  • Only the CR group {007, 008, 012, 017} is correctly assigned

Effect on paper claims. Every plot in Extended Data Figure 9 that uses x = "WHO" / fill = "WHO" (~10 panels from line 384 onward) compares two groups that are statistically identical mixtures of the same two true categories. By construction, any visible separation or printed Wilcoxon p-value on those PR-vs-SD comparisons is reading noise — not response biology. The CR group is correctly labeled, so CR-vs-anything contrasts are unaffected; the PR-vs-SD distinction in the WHO-stratified panels is structurally invalid. Headline pCR-vs-non-pCR contrasts elsewhere (which use Pathological.Response directly from metadata) are not on this code path.

Moderate findings

Finding 2 — Cell-barcode rename regex misses suffixes _10, _11, _12; ~9.6% of cells silently fail the clonotype join
VERIFIED NOVEL* MODERATE
Rule typeRegex anchor problem — greedy alternation truncates at one digit Locationnotebook_template/Figure4.Rmd:1018; Figure5.Rmd:283; ExtendedData_Figure10.Rmd:178

Cell barcodes from Seurat’s integration step carry a sample-index suffix of the form _1, _2, …, _12. The notebooks rename these back to plain barcodes before joining to the scRepertoire TCR/BCR table using a regex of the form (_1|_2|...|_9)$ — covering only the single-digit suffixes. Cells originating from samples 10, 11, and 12 keep their two-digit suffix (_10, _11, _12) and therefore do not match any row in the clonotype table.

Counting in the deposited barcode files: 10,801 cells across the three affected samples (NeoBCC10/11/12), or about 9.6% of the cohort, are silently excluded from every TCR/BCR-aware analysis — including Figure 4f and the clonality plots in Extended Data Figure 10.

Effect on paper claims. Sample-level TCR-clonality and BCR-clonality comparisons in Figures 4–5 and EDF10 use a smaller cohort than the paper text implies. NeoBCC12 is one of the four CR patients in the metadata; its omission from the TCR-aware figures changes the n on the CR side of those panels. The omission is silent because the affected cells simply have NA in the joined columns rather than throwing.

Finding 3 — Wilcoxon at the floor of the test; no FDR correction across ~50 comparisons
VERIFIED NOVEL* MODERATE
Rule typeTest ceiling / multiple-testing Locationstat_compare_means(method = "wilcox.test", ...) across Figures 3, 4, 5 and EDF6–10

The two-sided Wilcoxon rank-sum test has a hard lower bound on the achievable p-value as a function of the two group sizes. For the cohort splits used in the paper:

  • n = 4 vs n = 4 (PR vs SD, PR vs CR, SD vs CR — the only achievable splits in this 12-patient cohort): minimum two-sided p = 0.0286
  • n = 4 vs n = 8 (CR vs non-CR): minimum two-sided p = 0.0040

Figure 4d’s “CR vs SD P = 0.029” is exactly the floor: the test is reporting the smallest p-value the data shape can produce. The same holds for any other n=4 vs n=4 panel that prints P = 0.029. Across the manuscript and supplements, ~50 such Wilcoxon comparisons are computed; no false-discovery-rate correction is applied.

Effect on paper claims. A printed p-value at the test floor is not evidence of a real effect — it only requires that the two groups be perfectly rank-separable, which is structurally easier in a 4-vs-4 split than the p-value suggests. Combined with the absence of multiple-testing correction across ~50 such tests, the cumulative claim of repeated significance in this paper is overstated. Compounds with Findings 1 and 4 on the same Figure 4d panel.

Finding 4 — Cyto/Treg ratio in Figure 4d propagates NA silently; one missing patient drops the headline P = 0.029 below significance
VERIFIED NOVEL* MODERATE
Rule typeSilent NA propagation through a per-patient ratio Locationnotebook_template/Figure4.Rmd (per-patient cytotoxic/Treg ratio used by Figure 4d)

The cytotoxic-to-Treg ratio used in Figure 4d is computed per patient as Nb[anno_l2 == "CytoTcell"] / Nb[anno_l2 == "Tregs"]. When a patient’s subset has no Tregs detected, the denominator expression returns numeric(0), the ratio becomes NA, and the patient silently drops from the downstream Wilcoxon.

NeoBCC10 contributes only 133 cells (Finding 5) and is therefore the most likely patient to have zero detected Tregs on this panel. NeoBCC10 also lands in the “PR” group as analyzed (per the hardcode of Finding 1). If NeoBCC10 drops out of the Cyto/Treg panel, the “PR” group falls from n=4 to n=3, and the n=3 vs n=4 Wilcoxon floor is 0.057 — above the 0.05 threshold the paper claims to clear.

Effect on paper claims. The reported P = 0.029 on Figure 4d depends on every patient being present. Silent NA propagation makes the result fragile to a single-patient drop — and Finding 1 already shows that the “PR” and “SD” groups in this comparison are not the response groups they are labeled as. The combination means the panel is over-determined by group definition and under-determined by sample size.

Finding 5 — NeoBCC10 contributes 133 cells (~70× below cohort median) and enters Wilcoxon panels with full patient weight
VERIFIED NOVEL* MODERATE
Rule typePer-sample weighting — small-sample treated as full patient LocationPer-patient summary statistics consumed by Figures 3–5 and EDF6–10

Per-patient cell counts in the deposited cohort range from 133 (NeoBCC10) to 18,752, with a cohort median around 10,400. Every figure that aggregates to one value per patient (cell-fraction plots, clonality summaries, Cyto/Treg ratio) treats NeoBCC10’s 133 cells as carrying the same weight as the median patient’s ~10,000 cells. No QC threshold filters NeoBCC10 out, and no figure caption flags the disparity.

Per Finding 1, NeoBCC10 is also the patient that the hardcoded label assigns to PR rather than its true SD. So in the WHO-stratified n=4 Wilcoxons, a 133-cell sample is being used as one of four data points in a contaminated “PR” group.

Effect on paper claims. Any per-patient summary in Figures 3–5 and EDF6–10 is more sensitive to NeoBCC10’s small-sample noise than the equal-weight aggregation suggests. The effect is most acute in n=4 vs n=4 Wilcoxon tests, where one outlier patient is 25% of one group.

Finding 6 — McPAS join is alpha-only, lacks a Species filter, and uses order-dependent dedup
VERIFIED NOVEL* MODERATE
Rule typeReference-table join semantics Locationnotebook_template/Figure4.Rmd:1115–1131

The McPAS-TCR annotation join used to label cells as pathogen-reactive in Figure 4f does three things in combination:

  1. Alpha-only matching. Cells are matched to McPAS rows on CDR3.alpha.aa alone. McPAS entries are typically alpha+beta TCR pairs; a cell that matches a McPAS alpha but has a different beta is annotated as the McPAS pathology anyway. (Direct check: of the two cells labeled hHSV: in the rendered Figure 4f, one cell’s beta differs from the McPAS entry’s beta for the same alpha — the call is on partial-chain evidence only.)
  2. No Species filter. McPAS contains both Human and Mouse TCRs. A cell’s human alpha that happens to coincide with a mouse-TCR alpha in McPAS will inherit the mouse pathology label. The two herpes cells in Figure 4f happen to match Human McPAS rows; the protection is incidental, not coded.
  3. Order-dependent dedup. When a cell’s alpha matches multiple McPAS rows with different pathologies, !duplicated(joint$rid) picks the first row in McPAS file order. This is deterministic but unprincipled — a different McPAS row order would produce different pathology assignments for the same cells.

Effect on paper claims. The headline n=2 herpes-reactive cells in Figure 4f survive these issues for incidental reasons (both alphas resolve to a single Human McPAS row). For the broader pathogen-vs-cancer breakdown in EDF10, mouse rows can leak in for non-HSV pathologies, and ambiguous-alpha cells get a non-deterministic-from-the-paper’s-perspective label. The strength of any “X% of TCRs are pathogen-reactive” statement on this code path is weaker than the alpha-only match implies.

Finding 7 — Dead stat_compare_means comparison list in EDF10: factor levels are c("pCR","non-pCR") but the test references CR/PR/SD
VERIFIED NOVEL* MODERATE
Rule typeggpubr silent no-op — comparison labels not in the factor Locationnotebook_template/ExtendedData_Figure10.Rmd:1015 and 1024

The clonality plots g1 and g2 in EDF10 are constructed with x = "Response" where Response is a factor with levels c("pCR","non-pCR"):

q1$Response <- factor(q1$Response, levels = c("pCR", "non-pCR"))
g1 <- ggpaired(q1, x = "Response", ...,
               comparison = list(c("CR","SD"), c("SD","PR"), c("CR","PR")), ...)

The stat_compare_means comparison list references CR, SD, and PR — none of which are levels of the factor being plotted. ggpubr silently emits no significance annotations in this configuration. The fact that the panels render without bracket annotations is the only visible symptom; the missing comparisons are not flagged in the figure caption.

Because params$accessibility = "lock" by default, the entire EDF10 chunk is normally skipped on a clean clone — the bug only fires when a user with EGA-approved BCR data sets accessibility = "unlock", which masks it from any reader who does not have controlled-access data.

Effect on paper claims. Where EDF10 is reproduced from the unlocked code path, the “CR vs PR”, “CR vs SD”, and “SD vs PR” significance brackets the code attempts to draw simply do not appear. Any conclusion in the manuscript that points to those bracketed comparisons relies on output the unlocked code does not actually produce.

Reading guide

Findings 1, 3, 4, and 5 compound on the same Figure 4d / EDF9 panels. The PR-vs-SD comparisons there use groups that are 50/50 mixtures of the two true response categories (Finding 1), at sample sizes where Wilcoxon’s minimum two-sided p-value is the very value the paper reports as significant (Finding 3), with a small-sample patient (Finding 5) carrying full weight in one of those groups, and a silent NA propagation that can drop that patient from the test entirely and push the result above 0.05 (Finding 4). The structural conclusion is that the WHO-stratified Wilcoxon panels do not sustain the response-group claims they label.

The CR vs non-CR contrasts and the pCR-vs-non-pCR contrasts that read Pathological.Response directly from the metadata are not on the WHO code path and are not affected by Finding 1. Findings 2, 6, and 7 affect the TCR/BCR-aware figures and the Figure 4f herpes-reactivity panel; the headline n=2 herpes-reactive cells survive Findings 6 for incidental reasons (both alphas happen to resolve cleanly), but the broader pathogen-reactivity breakdown in EDF10 inherits the alpha-only and Species-filter weaknesses.

Findings rated MINOR or NEGLIGIBLE by the impact quantifier, plus reproducibility-only findings, are not reproduced here per scope. They include: a non-Ensembl HSV_genome row in gene_features.csv (T-VEC oncolytic-virus reference, not load-bearing for the paper’s claims); marker panels that mix HGNC symbols and protein aliases (CD25 / IL2RA, TIM3 / HAVCR2) which silently drop from dot plots; the absence of the manual cluster-annotation file Ressler2024_metadata.csv from the repository (a reproducibility issue that does not affect the published numbers); and the abstract’s pre/post p-values being computed on a separate mIF cohort (n=16) rather than on the public scRNA pipeline (a transparency issue affecting reproducibility from public materials only).
